CSS Transitions

First, let's look at CSS Transitions. These are a lot of fun and can really make for some cool effects like the example above where the sphere transitions from green to blue in 1 second. Here, we will use some similar animations and go over some of the properties that are related to transitions in CSS. The main properties that are modified in a CSS Transition are as follows:

transition
transition-delay
transition-duration
transition-property
transition-timing-function

These can all be expressed within a simple transform statement, or can be outlined in separate lines as shown above. If we were to compare the two, it might look like the following example:

div {
  transition-property: height;
  transition-duration: 5s;
  transition-timing-function: ease;
  transition-delay: 2s;
}

// is equivalent to:

div {
  transition: height 5s ease 2s;
}

Among these effects, the transitions can be customized further to create some truly awesome effects. The transition-timing-function, for example, comes with added customizations that allow the user to specify the speed curve of the transition effect. We will demonstrate some examples of some of these comparatively, but first, the values are as follows:

ease 
- specifies a transition effect with a slow start, then fast, then end slowly (this is default)

linear 
- specifies a transition effect with the same speed from start to end

ease-in 
- specifies a transition effect with a slow start

ease-out 
- specifies a transition effect with a slow end

ease-in-out 
- specifies a transition effect with a slow start and end

cubic-bezier(n,n,n,n) 
- lets you define your own values in a cubic-bezier function

So, let's see some of these in action! The code below is based loosely on a W3 Schools demonstration of transition timing functions. Notice when the transitioning objects slow down or speed up is different in each instance.

The transitions demonstrated above are the width and background color, which are both transitioned using the transition-property of all rather than specifying them individually for the duration specified for the animation. The syntax of that property can be any of the following:

transition-property: none|all|property|initial|inherit;

Also, notably, the transition-timing-function was applied outside of the transition statement and within the individual objects, further showing the versatility that these commands can have.

CSS Animations

The Last section we will discuss here will be about CSS Animations. Animations are a lot of fun and can be triggered in a number of different ways, from hover effects, to class changes, to onclick events. There are numerous ways to trigger and create them, which makes them extremely versatile and easy to work with. Animations can be defined with the following properties: 

@keyframes
animation-name
animation-duration
animation-delay
animation-iteration-count
animation-direction
animation-timing-function
animation-fill-mode
animation

Something that needs further clarification from the above list is the animation-fill-mode, which specifies a default value that an object can have while an animation is not running. This can be defined with the following values:

none 
- Default value. Animation will not apply any styles to the element before or after it is executing

forwards 
- The element will retain the style values that is set by the last keyframe (depends on animation-direction and animation-iteration-count)

backwards 
- The element will get the style values that is set by the first keyframe (depends on animation-direction), and retain this during the animation-delay period

both 
- The animation will follow the rules for both forwards and backwards, extending the animation properties in both directions

Also of note, these animations can also utilize the same timing-functions as above. This allows for animations to be defined with keyframes, but also allows for animation related timing to be defined. Rather than get into that, as I feel this is well represented above, let's get into a demonstration. You may have noticed that similar to transitions, the animation can be defined with one line or by specifying the properties on separate lines. For example:

div {
  animation-name: moveblock;
  animation-duration: 3s;
  animation-iteration-infinite: 2;
}

// is equivalent to:

div {
  animation: moveblock 3s infinite;
}
