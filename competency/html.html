LESSON 5


LOCAL STORAGE API, STORING AND RETRIEVING SIMPLE DATA, ARRAYS, ASSOCIATIVE ARRAYS, AND OBJECTS

So there's some good fun to be had with Local Storage API's, but first we need to understand what they are. There are primarily 2 variations of that storage. First, Local Storage (obviously), secondly Session Storage. Both Local Storage and Session Storage serve different purposes and can store up to 5 MB of text. That may not seem like much, but when you consider that 1 character is equal to 1 byte, you realize that 5MB is about 5 Million characters. When you consider that the average book has about 350 words a page (1500 to 1800 characters per page), that's only about 400,000 - 500,000 characters (bytes), on average, which is about a tenth of the storage that either Local or Session storage provide.

So, what is Session Storage, and how does it differ from Local Storage? Simply put, Session storage is information that is stored only during your current session of browsing. This could be something like your shopping cart on a website or field entries when registering for something. Local storage, on the other hand, is information that exists beyond your current session. This can be just about anything, but a good example might be the chrome function that auto fills your information or remembers your passwords when you allow it to.

It's important to know that the commands for Local Storage and Session Storage are completely interchangeable, which is nice, so as we go through I will introduce both of them, but focus entirely on only one of them for the sake of simplicity. It's also worth noting that this will tie in to some more JSON, so make sure you check that out first! So what are the commands? Let's have a look! 

(replace Local for Session if you need Session Storage)


//Storing Data methods

1. localStorage.setItem("key", "value");
2. localStorage["key"] = value;


//Example:

1. localStorage.setItem("Make", "Nissan");
2. localStorage["Make"] = "Nissan";


//Retrieving Stored Data

var myData = localStorage["key"];


//Example

var Make = localStorage["Make"]; 
//Not Nissan, notice the difference



//Removing Stored Data

localStorage.clear(); 
//Removes EVERYTHING! Not advisable

localStorage.removeItem("key");

You can also store strings using JSON Stringify as an alternative.

I know I said it in there, but I need to reiterate that localStorage.clear is NOT recommended. It's known as a scorched earth approach for a reason because it removes all stored local files, even ones you may not have wanted to remove. Also worth noting, you can store a string of variables using JSON Stringify, or you can store just one value, whatever you desire. For the purposes of our examples here we will only be storing individual values.

LESSON 6


DOM MANIPULATION USING CREATEELEMENT, APPENDCHILD, INSERTBEFORE, REMOVECHILD, ETC

Do you know what you are looking at right now? A DOM. The DOM of this page is pretty great right? objects that make up my layout are part of a structure, a Document Object Model. What you might not know though is that the DOM can be CHANGED! But what exactly does it mean?


It means that we can reorganize the objects on the layout! Or more than that, make new ones or remove them entirely!

Creating Elements

So let's start with Creating new Elements! It’s easy to create elements, all you need to do is something like this in JavaScript:

document.createElement("elementName");

//Example

document.createElement(“theFifth”);

So we create an element called "The Fifth", pun intended, but what now? “theFifth” element does not exist on our layout, we need to go ahead and add it somewhere

To do that, we need to do something like this

<script>
	var newElement = document.createElement(“theFifth”);
	newElement.textContent = “My name is a pun”;
	document.body.appendChild(newElement);
</script>

So, in case you didn't pick up on what just happened, this places our new element at the bottom of the Body element. The Fifth Element now exists as an element of our DOM. This image is based on a Kirupa tutorial, but has been changed slightly from theirs.


Removing Elements

Just as it is easy to add elements, it’s just as easy to remove them. To remove “theFifth” we simply need to do the following

document.body.removeChild(newElement);

You may notice that we used the variable name from our create element script rather than the element name, this is something worth remembering. Again, you can use this to remove objects that are native to the layout such as h1 etc. Removing an object, it also removes all of it’s children.


Cloning Elements

What if we wanted to clone an element? This is a bit more complicated, but not by much. It requires a little extra explanation. Let’s start by looking at this code: 

<div id="footer">
        <div class="share">
            <p>Something</p>
            <img alt="#" src="blah.png"/>
        </div>
</div>
 					
<script>
        var share = document.querySelector(".share");
        var shareClone = share.cloneNode(false); 
// False means only clone this

 
        document.querySelector("#footer").appendChild(shareClone);
</script>

It should look something like this if done correctly. Note, these changes don’t modify the original code. 

<div id="footer">
        <div class="share">
            <p>Something</p>
            <img alt="#" src="blah.png"/>
        </div>
        <div class="shareClone">
            <p>Something</p>
            <img alt="#" src="blah.png"/>
        </div>
</div>

Looking at this code again, we notice that the cloned element was added, or appended, to the footer using appendChild. Its uses are not limited to cloning, but otherwise is pretty self explanatory. 

<div id="footer">
        <div class="share">
            <p>Something</p>
            <img alt="#" src="blah.png"/>
        </div>
</div>
<script>
        var share = document.querySelector(".share");
        var shareClone = share.cloneNode(false);

The native function of insertBefore will allow you to move an element to anywhere you think might look best within the same parent. For example, if I wanted “theFifth” element to be placed before the heading, I would use the following code:

var newElement = document.createElement(“theFifth”);
newElement.textContent = “My name is a pun”;
 
var h1Element = document.querySelector(“h1”);
document.body.insertBefore(newElement, h1Element);
